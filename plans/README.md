## Findings considered and rejected

- **`/tabs` O(windowsГ—tabs) sequential `osa` calls (P-1)**: not worth a dedicated
  plan now вЂ” `/tabs` is a low-frequency read (tab listing), not a hot path, and
  batching the AppleScript into one `execute javascript`/multi-get would trade
  simplicity for a rare-case speedup. Revisit only if a caller reports latency
  with many tabs.
- **`fast-uri` transitive advisories (D-1)**: unreachable for routing (all
  routes are static path strings, not dynamic matchers), so the path-traversal/
  host-confusion issues can't be triggered through the bridge. A `pnpm update`
  is a free fix but not worth a plan; do it opportunistically with the next
  dependency bump.
- **`/key` returns `err.message` for unknown keys (S-2)**: it echoes only the
  caller's own `key` name (not server internals), so the "never leak err.message"
  rule's intent (no internal-detail leakage) isn't violated in spirit. Could be
  tightened to a fixed `"unsupported key"` + supported list (already attached);
  low value. Deferred вЂ” fold into 002's lint pass if desired.
- **`/navigate` URL validation accepts control chars в†’ 500 (C-3)**: the route
  already validates `^https?://`; a stray control char causing an AppleScript
  parse error surfaces as the standard 500 via `fail()` (no leak). Marginal
  hardening; not worth a plan.
- **`fail()` string-matches `(-1719)` (TD-2)**: works today; brittle only if
  AppleScript changes error wording. A structured error-code check would be
  cleaner but is a low-value refactor. Revisit if a macOS/Chrome update breaks
  the match.
- **`BROWSER` env interpolation**: host-owner == threat model; intentional.
- **`/eval` undefinedв†’`''` ambiguity**: by-design (documented in the route).
- **human-path `CGWarp` vs real mouse-move events**: feature-quality concern
  already covered in `docs/superpowers/specs/2026-05-17-human-mouse-move-design.md`;
  not a code-debt finding.
